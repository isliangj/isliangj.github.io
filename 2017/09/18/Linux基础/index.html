Linux 系统介绍

基本知识

  • 硬盘分为：IDE、SCSI和SATA硬盘。Linux中IDE接口的设备被称为hd，SCSI和SATA接口的设备被称为sd。第一块硬盘是sda，第二块为sdb……
  • Linux中一块硬盘只能存在4个主分区：sda1、sda2、sda3和sda4。
  • 主分区中sda4一般作为扩展分区（只能有一个），扩展分区不写入数据，只是包含逻辑分区，逻辑分区就不限制个数，标识从5开始。

与Windows的区别

  • Linux严格区分大小写（Linux是命令行 Windows是dos界面）
  • Linux中 一切皆文件，包括硬件、用户
  • Linux中 不靠扩展名区分文件类型，靠权限区分，但有约定俗称的扩展名
  • Linux中 不能直接安装和运行Windows的程序

分区

  1. 先分200M给/boot用于启动

  2. 后分2048M（两倍于内存大小但不超过2G）给swap交换分区

  3. 之后分1024M给/home存放登陆用户的配置和私人文件

  4. 最后把剩余的空间都分配给/根分区，存放系统所必须的文件

格式化

在磁盘的特定区域写入特定数据，在分区中划出一片用于存放文件分配表、目录表等用于文件管理的磁盘空间。

文件系统

  • FAT16（File Allocation Table 文件分配表）
  • FAT32、NTFS（New Technology File System）
  • EXT2（second extended filesystem第二代扩展文件系统）
  • EXT3
  • EXT4

快捷键

  • Ctrl + c   强制终止当前命令
  • Ctrl + l   清屏
  • Ctrl + a   光标移动到命令行首
  • Ctrl + e   光标移动到命令行尾
  • Ctrl + u   从光标所在位置删除到行首
  • Ctrl + z   把命令放在后台
  • Ctrl + r   在历史命令中搜索

目录结构

文件权限

-rw-rw-r–详解

  • 第一位: -:表示普通文件,d: 表示目录, l:表示符号链接
  • 第二到第十位: 如下所示:
  • -表示没有权限,r:表示读权限, w:表示写权限,x:表示执行权限

文件类型

  • 普通文件（-）： 包括文本文件、数据文件、可执行的二进制程序文件等。
  • 目录文件（d）： Linux系统把目录看成是一种特殊的文件，利用它构成文件系统的树型结构。
  • 设备文件： Linux系统把每一个设备都看成是一个文件

常用命令

文件处理命令

mkdir

  • 新建目录
  • 英文：mack directories
  • mkdir [选项] [目录名]
  • -p 递归创建

cd

  • 切换所在目录
  • 英文：change directory
  • cd [目录]
  • 简化操作：
    • cd ~ / cd    进入当前用户的家目录
    • cd -      进入上次目录
    • cd ..      进入上一级目录（加空格）
    • cd .       进入当前目录（无用）

ls

  • 显示当前目录内容
  • 英文：list
  • ls [参数] [路径或文件名]
  • 显示所有文件或目录（包含隐藏的文件）:ls -a
  • 显示详细信息：ls -l 缩写 ll
  • 友好显示文件大小：ll -h

du

  • 显示目录或文件的大小
  • du –h

cat

  • 查看整个文件内容
  • cat [参数] 文件名

more

  • 查看大文件一屏幕显示不完内容
  • 空格或者回车键显示下一行内容
  • 使用q或者ctrl+c退出

less

  • 省略大文件内容
  • 通过PgUp、PgDn键来控制

pwd

  • 查询所在目录位置
  • 英文：print working directory

rmdir

  • 删除空目录（少用）
  • 英文：remove empty directories
  • rmdir [目录名]

rm

  • 删除文件或目录（小心）
  • rm -rf [文件或目录]
  • 英文：remove
  • 选项：
    • -r   删除目录
    • -f   强制删除

touch

  • 创建空文件
  • touch 文件名

cp

  • 复制命令
  • cp [选项] [原文件或目录] [目标目录]
  • 英文：copy
  • 选项：
    • -r   复制目录
    • -p   连带文件属性复制
    • -d   若源文件是链接文件，则复制链接属性
    • -a   相当于 -pdr 所有的属性都一样

mv

  • 剪切或改名命令
  • 英文：move
  • mv [原文件或目录] [目标目录/新名字]
  • 注：剪切目录的时候不需要加 -r

wget

  • 下载文件
  • wget url

链接命令

ln

  • 链接命令
  • ln -s [原文件] [目标文件]
  • 英文：link
  • 功能描述：生成链接文件
  • 选项：
    • -s 创建软链接
  • 注：软链接的路径一定要写绝对路径
  • 硬链接：（不建议使用）
    • 可以看做是同一个文件
    • 可通过i节点识别
    • 不能跨分区
    • 不能针对目录使用
  • 软链接：
    • 类似于Windows的快捷方式
    • 拥有自己的i节点 和 block块，只保存原文件的文件名和i节点，并没有实际的数据
    • 修改任意文件，另一个都改变
    • 删除原文件，软链接不能使用

文件搜索命令

locate

  • 在后台数据库中按文件名搜索 搜索速度更快
  • locate 文件名
  • 后台数据库 /var/lib/mlocate （一天一更新）
  • 更新数据库命令 updatedb
  • 缺点：只能通过文件名搜索

whereis

  • 搜索命令的命令
  • whereis 命令名
  • 搜索命令所在路径及帮助文档的所在位置
  • 选项：
    • -b 只查找可执行文件
    • -m 只查找帮助文件

which

  • 搜索命令所在路径及别名
  • which 文件名

find

  • 搜索文件
  • find [搜索范围] [搜索条件]
  • find / -name install.log
  • 避免大范围搜索 会非常耗资源
  • find是搜索符合条件的文件名，如果需要模糊查询，使用通配符匹配通配符是完全匹配：
    • - 匹配任意内容
    • ? 匹配任意一个字符
    • 匹配任意一个中括号内的字符
  • 选项：
    • 不区分大小写 find /root -iname install.log
    • 按照所有者搜索 find /root -user root
    • 查找没有所有者的文件 find /root -nouser
    • 查找10天前修改的文件 find /var/log -mtime +10
    • 查找文件大小是25k的文件 find . -size 25k
    • 查找i节点是262422的文件 find . -inum 262422
    • 逻辑与 -a 逻辑或 -o
    • 对搜索结果在进行操作 -exec/-ok (命令) {} \

grep

  • 在文件中匹配符合条件的“字符串”
  • grep [选项] 字符串 文件名
  • 选项：
    • -i 忽略大小写
    • -v 排除指定字符串

帮助命令

man

  • 获取指定命令的帮助
  • man 命令
  • 英文：manual（文档）
  • 级别：
    • 0：关机
    • 1：单用户类似于win中的安全模式
    • 2：不完全多用户，不含NFS服务不包含NFS的普通字符界面
    • 3 ：完全多用户普通的字符界面
    • 4：未分配
    • 5：图像界面
    • 6：重启
  • 查看命令拥有哪个级别：man -f(级别) 命令 (相当于) whatis 命令
  • 查看和命令相关的所有帮助：man -k 命令 （相当于） apropos 命令

-help

  • 获取命令选项帮助
  • 命令 - -help

help

  • 获取shell内部命令的帮助
  • help 内部命令

info

  • 详细命令帮助（不常用）
  • info 命令
    • 回车：进入子帮助页面（带有*号标记）
    • u：进入上层页面
    • n：进入下一个帮助小节
    • p：进入上一个帮助小节
    • q：退出

压缩和解压缩命令

.zip

  • 压缩文件： zip 压缩文件名 源文件
  • 压缩目录：zip -r 压缩文件名 源目录
  • 解压缩文件：unzip 压缩文件

.gz

  • 压缩后源文件消失：gzip 源文件
  • 压缩后源文件保留：gzip -c 源文件 > 压缩文件
  • 压缩目录下的所有子文件，但不压缩目录：gzip -r 目录
  • 解压：gzip -d 压缩文件 / gunzip 压缩文件

.bz2

  • 压缩后不保留源文件：bzip2 源文件
  • 压缩后保留源文件：bzip2 -k 源文件
  • 解压（-k保留压缩文件）：bzip2 -d 压缩文件 / bunzip2 压缩文件
  • 注：bzip2不能压缩目录

tar

  • 打包
  • tar -cvf 打包文件名 源文件
  • 选项：
    • -c：打包
    • -v：显示过程
    • -f：指定打包后的文件名
  • 打包之后再压缩
  • 解打包命令：tar -xvf 打包文件名
  • 选项：
    • -x：解打包

.tar.gz（常见）

  • 先打包 再压缩/解压
  • 压缩：tar -zcvf 压缩包名.tar.gz 源文件
  • 选项：
    • -z：压缩为.tar.gz格式
  • 解压缩：tar -zxvf 压缩包名.tar.gz
  • 选项：
    • -x：解压缩.tar.gz格式
  • 指定位置解压：tar -zxvf 压缩包名.tar.gz -C 解压目录
  • 查看压缩包：tar -ztvf 压缩包名.tar.gz

.tar.bz2（常见）

  • 先打包 再压缩/解压
  • 压缩：tar -jcvf 压缩包名.tar.bz2 源文件
  • 选项：
    • -z：压缩为.tar.bz2格式
  • 解压缩 tar -jxvf 压缩包名.tar.bz2
  • 选项：
    • -x：解压缩.tar.bz2格式
  • 指定位置解压：tar -jxvf 压缩包名.tar.bz2 -C 解压目录
  • 查看压缩包：tar -ztvf 压缩包名.tar.bz2

关机和重启命令

shutdown

  • shutdown [ 选项 ] 时间
  • 选项：
    • -c：取消前一个关机命令
    • -h：关机
    • -r：重启

  • 时间：

    • now：立即重启
    • 00：00 定时重启

  • 其他关机命令（不安全）：

    • halt
    • poweroff
    • init 0

  • 其他重启命令：

    • reboot
    • init 6 （不安全）

  • 退出登录：logout

网络通讯命令

ifconfig

  • 显示或设置网络设备
  • 显示网络设备：ifconfig
  • 启用eth0网卡：ifconfig eth0 up
  • 停用eth0：ifconfig eth0 down

ping

  • 测试网络状态
  • ping 192.168.164.1

netstat

  • 查看网络端口
  • 选项：
    • -t : 指明显示TCP端口
    • -u : 指明显示UDP端口
    • -l : 仅显示监听套接字(所谓套接字就是使应用程序能够读写与收发通讯协议(protocol)与资料的程序)
    • -p : 显示进程标识符和程序名称，每一个套接字/端口都属于一个程序。
    • -n : 不进行DNS轮询(可以加速操作)

hostname

  • 查看当前主机名

uname

  • 显示系统信息
  • uname -a 显示本机详细信息。 依次为：内核名称(类别)，主机名，内核版本号，内核版本，内核编译日期，硬件名，处理器类型，硬件平台类型

挂载命令

mount

  • mount [ -t文件系统 ] [-o 特殊选项] 设备文件名 挂载点
  • 选项：
    • -t 文件系统：加入文件系统类型来指定挂载的类型。可以ext3、ext4、ISO9660 等文件系统
    • -o特殊选项：可以指定挂载的额外选项

  • 查询系统中已经挂载的设备：mount

  • 依据配置文件/etc/fstab的内容 自动挂载：mount -a

  • 注：LInux 挂载点 == Windows 分配盘符
  • 建立挂载点：mkdir /mnt/cdrom/

  • 挂载光盘

    • mount -t iso9660 /dev/cdrom /mnt/cdrom/

    • mount /dev/sr0 /mnt/cdrom

  • 卸载命令：umount 设备文件名或挂载点
  • 注：先退出光盘的目录；一定要先卸载才能拿出光盘
  • 挂载U盘：mount -t vfat /dev/sdb1 /mnt/usb
  • 查看U盘设备文件名：fdisk -l
  • 注：LInux默认不支持NTFS文件系统

Vim

操作模式：

  • 输入模式：“i” 切换
  • 命令模式：默认的模式；用于输入命令；输入命令，不能查看，立即执行
  • 底行模式：“esc” 切换 ；“:wq” 保存退出；输入命令，底行查看，回车执行

创建文件

vim “文件名”

  • 创建并打开文件

vim + “文件名”

  • 打开文件后 光标定位到最后一行

vim +3 “文件名”

  • 打开文件后 光标定位到第三行

vim +/imooc “文件名”

  • 打开文件 光标定位到imooc第一次出现的行
  • 按 “n” 切换下一个imooc出现的地方

vim “文件名1” “文件名2” “文件名3”

  • 创建多个文件 并打开第一个文件
  • 在底行模式输入 “:n” 切换下一个文件
  • 底行模式 “:N” 或 “:prev” 切换上一个文件

底行模式

:wq 保存并退出

:w保存

:q退出文件

:! 强制执行

:!q 不保存 并退出

:ls 列出打开的所有文件

:n切换后一个文件

:N 切换前一个文件

:3 光标定位到第三行

/xxx 从光标定位开始向后搜索“xxx”第一次出现的行

?xxx 从光标定位开始向前搜索“xxx” 第一次出现的行
