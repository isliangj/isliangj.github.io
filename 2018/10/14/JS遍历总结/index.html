JS遍历总结

2018年10月14日 | JS 笔记

for

  • 适用：数组和字符串
  • 不使用：对象
  • 改进的写法：

for-in

  • 适用：字符串、对象、数组
  • 不使用：Set 、 Map
  • 定义：以任意顺序遍历一个对象的可枚举属性
  • 遍历数组注意：它不仅遍历数组中的元素，还会遍历自定义的属性，甚至原型链上的属性都被访问到。而且，遍历数组元素的顺序可能是随机的。
  • 所以它更适合遍历对象，不适合遍历数组，但是适合遍历稀疏数组
  • 遍历速度慢，如果需要遍历一个数量有限的已知属性列表，使用其他循环会更快

forEach

  • 适用：数组、Set、Map
  • 不适用：字符串、对象
  • 按升序为数组中含有效值的每一项执行一次callback 函数，那些已删除或者未初始化的项将被跳过（例如在稀疏数组上）
  • 会遍历完所有的元素，不能 break 和 return
  • 在原数组上进行修改
  • 返回值：undefined

for-of

  • 适用：字符串、数组、Set、Map
  • 不使用：对象，需要实现Symbol.interator接口
  • ES6新增
  • 可以正确响应 break, continue, return

map

  • 适用：数组
  • 返回：新数组，且函数必须有返回值
  • 将原数组中的每个元素都按顺序调用一次 callback 函数。callback 每次执行后的返回值（包括 undefined）组合起来形成一个新数组。

reduce

  • 适用：数组
  • 让数组中的前项和后项做某种计算,并累计最终值
  • 返回值：函数累计处理的结果

filter

  • 适用：数组
  • 返回：不会改变原数组，它返回过滤后的新数组
  • 利用所有使得 callback 返回 true 或 等价于 true 的值 的元素创建一个新数组

every

  • 适用：数组
  • every 不会改变原数组
  • 当所有的元素都符合条件才返回true。另外，空数组也是返回true
  • 如果发现了一个为 false 的元素，every 方法将会立即返回 false

some

  • 适用：数组
  • some 不会改变数组
  • 数组中的每一个元素执行一次 callback 函数，直到找到一个使得 callback 返回一个“真值”（即可转换为布尔值 true 的值）。如果找到了这样一个值，some 将会立即返回 true。否则，some 返回 false。另外，空数组返回 false

find

  • 适用：数组
  • 对数组中的每一项元素执行一次 callback 函数，直至有一个 callback 返回 true。当找到了这样一个元素后，该方法会立即返回这个元素的值，否则返回 undefined。
